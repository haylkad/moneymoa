MOA. 설치 가이드

현재 상태(앱-시트 연동 완료) 기준, 최신 버전으로 가는 최종 절차.
위에서 아래로 순서대로 — 순서 중요해요 (비서 연결은 재배포 뒤!).

⏱️ 설치 15~20분 · 아래쪽에 운영 가이드(멀티 디바이스·백업·루틴) 포함

본 설치 — 순서대로

1
Apps Script 코드 교체
📍 구글시트 → 확장 프로그램 → Apps Script
  1. 기존 코드 전체 선택 → 삭제 후 새 Code.gs 내용 붙여넣기
  2. 코드 상단 상수 입력:
    TG_TOKEN = 봇 토큰 (@BotFather → /newbot, MOA 전용 봇 추천)
    TG_CHAT_ID = 내 챗ID (@userinfobot에게 아무 말 보내면 알려줌)
    GEMINI_KEY = (선택) aistudio.google.com/apikey 무료 발급
  3. 💾 저장 (Ctrl+S)
⚠️ 잊지 말기: 새 봇이면 텔레그램에서 봇에게 /start를 먼저 보내둬야 메시지를 받을 수 있어요.
2
시트 탭 보강 — setupSheets ▶
📍 Apps Script 상단 함수 드롭다운 → setupSheets 선택 → ▶ 실행
처음 실행하면 권한 창이 떠요: 권한 검토 → 계정 선택 → 고급 → "안전하지 않은 페이지로 이동" → 허용. 내 스크립트라 안전합니다.

실행하면 없는 탭만 새로 생겨요 (저축·고정지출·투자금·설정). 기존 데이터는 안 건드립니다.
3
자동화 트리거 3종 — 각 1회 ▶
📍 같은 곳, 함수 드롭다운에서 하나씩 선택 → ▶
  1. setupBackupTrigger — 매일 04시 시트 백업 (드라이브 MOA_백업, 30일 보관)
  2. setupFixedExpenseTrigger — 매일 06시 고정지출 자동 기입 (실행 즉시 밀린 것 처리)
  3. setupBriefingTrigger — 매일 08시·21시 텔레그램 브리핑 (실행 직후 테스트 메시지 1건이 와야 정상)
💡 시간 바꾸려면 함수 안의 atHour(8) 숫자만 고치고 다시 ▶.
4
새 버전 재배포 — 가장 많이 빠뜨리는 단계
📍 Apps Script 우상단 배포 → 배포 관리 → ✏️ 연필
  1. 버전: "새 버전" 선택
  2. 액세스: "모든 사용자" 확인 — "Google 계정이 있는 모든 사용자" 아님! (둘이 달라요)
  3. 배포 → URL은 기존과 동일하게 유지 → 앱 재연결 불필요
⚠️ 코드만 저장하고 재배포 안 하면 앱·봇은 옛날 코드로 돌아갑니다. "분명 고쳤는데 그대로네?"의 90%가 이거예요.
5
텔레그램 비서 연결 — setupTelegramAssistant ▶
📍 함수 드롭다운 → setupTelegramAssistant → ▶ (반드시 4번 재배포 후에!)
텔레그램으로 "🤝 비서 연결 완료!"가 오면 성공. 이제 봇에게 금액을 보내면 구매 가능 여부를 답해줍니다.
💡 메시지가 안 오면: 코드 상단 WEBAPP_URL에 배포 관리의 /exec URL을 직접 붙여넣고 저장 → 다시 ▶.
6
앱 파일 교체 (GitHub Pages)
📍 GitHub repo
최신 moa.html을 index.html로 이름 바꿔 업로드(덮어쓰기) → 1~2분 후 반영. 로컬로 쓴다면 python -m http.server 폴더의 파일만 교체.

그다음 앱 설정 → 📲 다른 기기 연결 링크 복사 → 카톡 나에게 보내기 → 폰에서 열면 자동 연결 → 폰 홈 화면에 추가.
7
초기 데이터 입력 — 앱에서 (10분)
📍 MOA 앱 ＋ 버튼
  1. 고정지출 등록 (가계부 ＋ → 고정 등록): 월세·구독·통신비·공과금. 금액이 매달 다른 공과금은 예상치 + 자동기록 Y
  2. 투자금 (투자 ＋ → 투자금 입금): 이번 달 월초에 빼놓은 적립금
  3. 위시리스트 (홈 ＋): 진짜 갖고 싶은 것들, 3만~100만 가격대 섞어서
  4. 6/30 월말: 홈 ＋ → 자산 스냅샷 — 추정치가 미리 채워져 있으니 실제 잔고와 비교해 저장

✅ 완료 체크 — 전부 통과하면 끝

☐ 텔레그램에 테스트 브리핑 도착 (3단계 직후)
☐ 봇에게 35000 사도됨? → 판정 답장 옴
☐ 🤔/🚫 받고 위시 → 시트 위시리스트 탭에 등록됨
☐ (키 입력했다면) 맥스 이번 달 진단해줘 → 👑 MAX 답변
☐ 앱 가계부에 고정지출 체크 카드 ✅/🕐 표시
☐ 앱 가계부 캘린더에 일별 금액 표시
☐ 폰·PC 양쪽에서 같은 데이터 확인

💻 운영 1 — 모든 기기에서 쓰기 (멀티 디바이스)

처음 한 번 — GitHub Pages에 올리기 (아직 안 했다면, 5분):
  1. github.com → New repository → 이름 예: moa → Public → Create
  2. Add file → Upload files → moa.html을 index.html로 이름 바꿔 업로드 → Commit
  3. Settings → Pages → Branch: main → Save
  4. 1~2분 뒤 https://아이디.github.io/moa 가 내 앱 주소. 이후 업데이트는 index.html만 덮어쓰면 끝
기기 추가 — 어떤 컴퓨터든 30초:
  1. 이미 연결된 기기에서: 설정 → 📲 다른 기기 연결 링크 복사 → 카톡 나에게 보내기
  2. 새 기기에서 그 링크 열기 → 자동 연결 끝. (링크 없이면: 앱 주소 접속 → 설정에 Apps Script URL 붙여넣기)
  3. 폰은 홈 화면 추가까지: 아이폰 Safari 공유 버튼 → "홈 화면에 추가" / 안드로이드 Chrome ⋮ → "홈 화면에 추가"
동기화는 자동: 모든 기기가 같은 시트를 읽고 쓰므로 폰에서 입력 → PC에서 보임. 앱으로 돌아오면 자동 새로고침되고(60초 스로틀), 급하면 우상단 🔄. 저축률 같은 설정도 시트로 동기화돼요.
⚠️ moa.html을 더블클릭으로 여는 file:// 방식은 연결이 막혀요. 반드시 GitHub Pages 주소나 python -m http.server로 띄운 localhost에서 쓰세요.

🗄️ 운영 2 — 백업과 복원

3중 안전망이 자동으로 돌아갑니다 (설치 3단계에서 켠 것):
  1. 매일 04시 자동 백업 — 구글 드라이브 MOA_백업 폴더에 시트 사본, 최근 30일치 보관
  2. 구글시트 버전 기록 — 시트 자체가 모든 수정을 기록 (파일 → 버전 기록 → 이전 버전 보기)
  3. 드라이브 휴지통 — 시트를 통째로 지워도 30일간 복구 가능
복원 방법 — 상황별:
  1. 입력 실수 (오타·중복) → 시트에서 해당 행 직접 수정/삭제 → 앱 새로고침. 끝
  2. 실수로 많이 지움 → 파일 → 버전 기록 → 사고 전 시점 선택 → "이 버전 복원"
  3. 대형 사고 (탭 삭제 등) → 드라이브 MOA_백업에서 어제 사본 열기 → 망가진 탭 내용을 본편 시트에 복사해 넣기 (사본을 본편으로 바꾸면 Apps Script를 다시 묶어야 해서 비추)
💡 백업이 잘 쌓이는지 설치 다음 날 드라이브에서 MOA_백업 폴더 한 번만 확인해보세요. 그 뒤로는 잊고 살아도 됩니다.

📆 운영 3 — 일상 루틴 (이것만 하면 됨)

주기할 일
매일 (1분)쓸 때마다 가계부 ＋로 지출 입력. 살까 말까는 봇에게 금액 먼저 (35000 사도됨?). 8시·21시 브리핑은 읽기만
월초 (2분)투자 ＋ → 투자금 입금 기록 · 가계부의 고정지출 체크 카드에서 ✅ 자동 기입 확인
월말 (3분)홈 ＋ → 자산 스냅샷: 추정치가 채워져 있으니 실제 잔고와 비교해 저장 → 그래프·연속기록 갱신
수시적금 부으면 목표 ＋ → 저축 입력 · 갖고 싶은 건 봇에 "위시" 또는 홈 ＋
수정·삭제전부 시트에서 직접 (앱·봇은 추가 전용). 고정지출 해지도 시트에서 행 삭제
안 해도 되는 것백업 · 고정지출 기입 · 기기 동기화 · 브리핑 — 전부 자동
